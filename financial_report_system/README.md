## 一、系统能做什么

### 核心能力

| 能力 | 说明 |
|------|------|
| **多文件上传** | 支持同时上传多份财务报表文件（如年报 + 季报），系统自动识别期间、跨期对比 |
| **多 Sheet 解析** | 自动识别每份文件中的所有有效工作表（资产负债表、利润表、现金流量表等），无需手动拆分 |
| **双引擎读取** | `.xls`（xlrd）和 `.xlsx`（openpyxl）两种 Excel 格式均支持 |
| **期间类型识别** | 自动判断年报 / Q1-Q3 季报 / 月报，并对季报流量指标进行年化折算 |
| **六维风险评分** | 偿债能力、盈利能力、现金流质量、营运能力、税务合规、财务造假预警 |
| **Beneish M-Score** | 完整 8 因子模型，逐因子展示计算详情 |
| **知识库规则引擎** | 16 条风控规则自动触发，附监管依据和行业基准 |
| **三表勾稽校验** | 期末现金一致性、会计恒等式、利润含金量自动验证 |
| **HTML / PDF 报告** | 生成专业授信决策报告，含雷达图、趋势图、行业基准对照 |
| **异步任务** | 支持长时间分析任务的进度轮询，前端实时展示 |

### 分析的六个维度

```
┌──────────────────────────────────────────────────────────────┐
│                    六维风险评分体系                            │
├──────────────┬───────────────────────────────────────────────┤
│ 偿债能力     │ 流动比率、速动比率、资产负债率、利息保障倍数    │
│ 盈利能力     │ 毛利率、净利率、ROE、ROA                      │
│ 现金流质量   │ 经营现金流正负、利润含金量、现金比率            │
│ 营运能力     │ 应收账款周转天数、存货周转天数、资产周转率      │
│ 税务合规     │ 增值税差异、实际所得税率                       │
│ 造假预警     │ Beneish M-Score 8因子、跨期异常检测            │
└──────────────┴───────────────────────────────────────────────┘
```

每个维度采用 **Sigmoid 平滑评分**（0-100 分），避免硬阈值边界突变。综合评分加权汇总后映射为三级九等信用评级（AAA ~ CCC）。

## 二、输入文件的处理方式

### 支持的文件格式

| 格式 | 扩展名 | 引擎 | 说明 |
|------|--------|------|------|
| Excel 97-2003 | `.xls` | xlrd | 旧格式，仅支持格式化值（不支持公式求值） |
| Excel 2007+ | `.xlsx` | openpyxl | 推荐格式，支持 `data_only=True` 读取公式结果 |

### 多文件识别

系统支持同时上传 **多份文件**（如 `2025年年报.xlsx` + `2026年Q1季报.xlsx`）：

- 自动按文件名中的日期排序，确定最新期和对比期
- 对季报的流量指标（收入、利润等）自动乘以年化因子，确保跨期可比性
- 生成跨期趋势分析（资产增长率、收入增速、利润增速、负债率变动）

### 多 Sheet 解析

每份 Excel 文件中可能包含 **多张工作表**，系统会自动识别并提取有效表格：

| Sheet 名称关键词 | 识别为 | 提取内容 |
|-----------------|--------|---------|
| 含"资产负债" | 资产负债表 | 资产、负债、所有者权益各科目 |
| 含"利润" | 利润表 | 收入、成本、费用、利润各科目 |
| 含"现金流"或"资金流量" | 现金流量表 | 经营/投资/筹资活动现金流 |
| 其他 | 跳过或标记为"其他" | 不参与核心分析 |

**示例**：一份 `.xls` 文件包含 6 个 Sheet：

| Sheet 名称 | 识别结果 | 行×列 |
|-----------|---------|-------|
| 其他 | 跳过 | 0×0 |
| 资产负债表 | ✅ 资产负债表 | 44×10 |
| 利润及利润分配表 | ✅ 利润表 | 47×6 |
| 现金流量表 | ✅ 现金流量表 | 39×6 |
| 表外数据录入 | 跳过 | 63×5 |

系统从 3 张有效 Sheet 中提取财务字段，汇总为扁平化的指标字典用于后续分析。

### 期间类型自动识别

| 月份 | 期望本期/累计比 | 判定结果 | 年化因子 |
|------|----------------|---------|---------|
| 12月 | < 20% | 年报 | 1.0（不年化） |
| 3月 | 25% ~ 70% | Q1 季报 | ×4.0 |
| 6月 | 35% ~ 70% | Q2 季报 | ×2.0 |
| 9月 | 60% ~ 95% | Q3 季报 | ×1.333 |
| 其他 | — | 月报 | ×12/月份 |

## 三、任务边界（能做 vs 不能做）

### ✅ 系统能做的

- 解析标准中国企业会计准则格式的三表（资产负债表、利润表、现金流量表）
- 识别年报、季报（Q1/Q2/Q3）、月报，并自动年化折算
- 对同一企业的多期报表进行跨期对比分析
- 根据六维评分模型生成信用评级和授信建议
- 通过 Beneish M-Score 模型检测财务造假风险
- 生成包含雷达图、趋势图、三表原文的 HTML/PDF 报告
- 支持 6 种行业的差异化权重配置

### ⚠️ 有限支持

- **非标准格式报表**：仅支持关键词匹配的科目名称，非标准命名的科目可能无法识别
- **合并报表 vs 单体报表**：系统不做区分，以 Sheet 中实际数据为准
- **外币报表**：不做汇率折算，直接使用报表中的人民币金额
- **现金流量表缺失**：当文件中无现金流量表时，系统用间接法估算经营现金流（精度有限）

### ❌ 系统不能做的

- 不做行业尽职调查或企业背景核查（需配合企查查等工具）
- 不做贷款审批决策（报告仅供信贷委员会参考）
- 不支持 PDF 格式的财务报表（仅支持 .xls/.xlsx）
- 不支持银行、证券等金融机构的专用报表格式
- 不做实时股价或市场数据接入
- 不做财务预测或情景模拟

## 四、输出报告内容

生成的 HTML 报告包含以下板块（按顺序）：

| 板块 | 内容 |
|------|------|
| 封面 | 企业名称、行业、信用等级、综合评分、授信建议 |
| 核心指标 | 资产总额、营业收入、净利润、流动比率、资产负债率、ROE 等 |
| 六维评分雷达 | 雷达图 + 各维度得分及权重 |
| 三表勾稽校验 | 期末现金一致性、会计恒等式、利润含金量验证结果 |
| Beneish M-Score | 总分 + 8 因子逐项详情（DSRI/GMI/AQI/SGI/DEPI/SGAI/TATA/LVGI） |
| 输入文件清单 | 各文件的 Sheet 列表（名称、类型、行×列） |
| 资产负债表 | 原始数据三栏展示（资产/负债/权益） |
| 利润表 & 现金流量表 | 并排两栏展示，关键行高亮 |
| 行业基准对照 | 与同行业参考值对比 |
| 风险预警清单 | 知识库触发的规则汇总（含监管依据） |
| 六维度详细分析 | 各指标的值、得分、触发规则 |
| 跨期趋势分析 | 两期数据对比（资产增长、收入增速、利润增速等） |
| 知识库评判 | 每条触发规则的详细说明（监管依据、会计准则、授信建议） |
| 信用评级说明 | 三级九等评级体系说明 |

## 五、快速开始

### Docker 部署（推荐）

```bash
git clone https://github.com/dugd-insight/credit-risk-analysis-system.git
cd credit-risk-analysis-system
cp .env.example .env
docker compose up --build -d
# 访问 http://localhost:8000
```

### 本地运行

```bash
# 安装依赖
pip install -r requirements.txt

# 启动服务
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
# 访问 http://localhost:8000
```

### 命令行（CLI）

```bash
cd risk_engine

# 单期分析
python main.py --files ../data/2025.12.xlsx --company "企业名称" --industry 制造业

# 多期对比
python main.py --files ../data/2025.12.xlsx ../data/2026.3.xlsx --company "企业名称" --industry 担保/金融服务

# 扫描目录
python main.py --dir ../data --company "企业名称" --industry 制造业
```

## 六、API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/` | GET | Web 上传页面 |
| `/health` | GET | 健康检查 |
| `/api/analyze` | POST | 同步分析（等待完成） |
| `/api/analyze-async` | POST | 异步分析（返回 task_id） |
| `/api/task/{task_id}` | GET | 查询任务进度 |
| `/api/report/{filename}` | GET | 下载报告 |
| `/api/batch-analyze` | POST | 批量分析（指定目录） |
| `/api/system/status` | GET | 系统状态 |
| `/docs` | GET | Swagger API 文档 |

### 调用示例

```python
import requests

files = [
    ('files', ('年报.xlsx', open('2025年报.xlsx', 'rb'))),
    ('files', ('季报.xlsx', open('2026Q1.xlsx', 'rb'))),
]
data = {
    'company_name': '某某公司',
    'industry': '制造业',
}

resp = requests.post('http://localhost:8000/api/analyze', files=files, data=data)
result = resp.json()
print(f"评分: {result['total_score']}, 等级: {result['grade']}")
print(f"报告: {result['report_url']}")
```

## 八、信用评级体系

| 等级 | 分数区间 | 授信建议 |
|------|---------|---------|
| AAA | 90-100 | 建议足额授信 |
| AA | 80-89 | 建议正常授信 |
| A | 70-79 | 审慎授信 |
| BBB | 60-69 | 附条件授信 |
| BB | 50-59 | 压缩授信额度 |
| B | 40-49 | 建议拒绝或要求强担保 |
| CCC | 0-39 | 建议拒绝授信（一票否决） |

> **一票否决机制**：当 Beneish M-Score > -1.78 时，无论其他维度得分如何，直接评定为 CCC。

## 九、知识库规则概览

内置 16 条风控规则，覆盖六大维度：

| 类别 | 规则数 | 示例规则 |
|------|--------|---------|
| 偿债能力 | 5 条 | 流动比率 < 1.0 → 高风险（银保监流动资金贷款管理办法） |
| 盈利能力 | 3 条 | ROE < 3% → 低盈利能力（银保监借款人评价） |
| 现金流 | 3 条 | 净利润含金量 < 50% → 利润真实性存疑 |
| 营运能力 | 2 条 | 应收账款周转天数 > 180天 → 回款风险 |
| 税务合规 | 2 条 | 增值税税负率 < 0.5% → 税务异常 |
| 造假预警 | 1 条 | M-Score > -1.78 → 财务造假高度怀疑 |

每条规则包含：触发条件、风险等级（CRITICAL/HIGH/MEDIUM/LOW）、监管依据、适用会计准则、行业基准、授信处置建议。
